## üìã Contribution Guidelines

### 1. Fork & Clone

- Fork the repository to your own GitHub account.
- Clone your fork locally:
  ```bash
  git clone https://github.com/open-devhub/quillbot.git
  cd quill-bot
  ```
  ### 2. Create a Branch
- Use a descriptive branch name:
  ```bash
  git checkout -b feature/add-math-parser
  ```

### 3. Make Changes

- Keep code style consistent with the project.
- Add comments where necessary.
- Update documentation if your changes affect usage.

### 4. Test Your Changes

- Ensure your code compiles and runs correctly.
- Add or update tests if applicable.
- Verify that commands (;compile, ;suggest, /math-breakdown) still work as expected.

### 5. Commit

- Write clear, concise commit messages:
  ```bash
  git commit -m "feat: add step-by-step math parser"
  ```

### 6. Push & Pull Request

- Push your branch to your fork:

  ```bash
  git push origin feature/add-math-parser
  ```

- Open a Pull Request (PR) against the main branch of the original repo.
- Describe your changes clearly in the PR description.
